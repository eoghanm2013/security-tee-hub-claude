# Ideas

Future ideas and explorations for the Security TEE Hub.

## Structure

Each idea gets its own folder with:
- `README.md` -- Overview and status
- Supporting docs, mockups, architecture diagrams

## Current Ideas

| Idea | Status | Description |
|------|--------|-------------|
| [TEE Hub Web App](./tee-hub-web-app/) | Concept | Standalone web app for investigations |
| [Knowledge Flywheel](./knowledge-flywheel/) | Concept | Auto-extract patterns from investigations into a living knowledge base |
| [Auto-Triage Pipeline](./auto-triage-pipeline/) | Concept | Flip the investigation model so the system researches first, TEE reviews |
| [Team Knowledge Surface](./team-knowledge-surface/) | Concept | Make investigation patterns and docs accessible to all TEEs |
| [Sandbox Regression Suite](./sandbox-regression-suite/) | Concept | Run [sandbox suite](https://github.com/eoghanm2013/security-sandbox-suite) continuously to catch security product regressions before customers |
| [Engineering Feedback Loop](./engineering-feedback-loop/) | Concept | Track escalation-to-fix lifecycle and build metrics |

## Adding New Ideas

```
ideas/
├── your-idea-name/
│   ├── README.md      # Overview, goals, status
│   ├── architecture.md # Technical design (if applicable)
│   └── mockups/       # UI sketches (if applicable)
```
